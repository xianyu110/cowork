Cowork 与普通对话的区别

📁

在文件环境中工作

直接在你的文件夹中运行，读取、整理和创建文件。不只是生成内容，而是在你的实际工作环境中操作

🤖

启动子代理

将复杂任务分解给独立的子代理并行处理，每个都有全新的上下文。突破单线程限制，处理更大规模的工作

⏱️

交付后查看

处理任务编排、上下文管理和进度跟踪。你可以随时查看进展，或让它自主运行直到完成

Cowork 能做什么

Cowork 在你希望 Claude 采取行动并执行任务时最为出色。无论是在环境中操作文件、处理大规模研究，还是生成完整的交付成果。

📂 访问本地文件

指向你机器上的文件夹：笔记、项目目录、研究档案。Claude 直接读取、创建和整理文件

"整理我的下载文件夹，按类型和项目分组"

"找出可以删除的重复照片和截图"

📊 创建深度工作成果

通过跨工具研究、综合信息和迭代输出，生成大型幻灯片、电子表格和报告

"我要去里斯本一周。搜索邮箱和日历获取行程详情，搜索天气、社区、餐厅和景点，创建行程文档"

"我在 Teams 和 Zendesk 上有 50+ 条客户反馈。找出主要投诉，然后搜索 SharePoint 中的文档开始起草回应"

⚡ 完成长时间运行的任务

在普通对话中会中断的工作（达到上下文限制、中途失去线索）在 Cowork 中可以运行到完成

"浏览整个笔记库，找出孤立的笔记、错过的联系，建议更好的文件夹结构"

"阅读 /contracts 中的所有文档，为每个创建关键条款、续约日期和义务的摘要"

🔄 并行处理复杂工作

当任务有独立部分时，Claude 可以启动子代理同时处理并综合结果

"我在评估四个供应商。启动子代理研究每个的定价、支持声誉和集成选项，给我对比"

"用子代理从三个角度分析这个决定：财务影响、客户体验和运营风险，综合推荐"

🌙 异步工作

启动任务，离开，回来看到完成的工作。想查看进度时查看，或让 Claude 自主运行

"周五有绩效评估。搜索 Slack、Google Drive 和 Asana 查看完成的任务、项目更新、同事反馈，起草会议准备表"

"查看上季度的团队会议记录和 Slack 频道。我的团队取得了什么成就？"

何时使用 Cowork

✅ 使用 Cowork

  • 需要在文件系统中实际操作
  • 任务涉及多个步骤或数据源
  • 需要创建具体的交付成果
  • 工作可能超出单个对话的上下文限制
  • 可以分解为并行子任务

💬 使用普通对话

  • 快速问答和头脑风暴
  • 需要实时来回讨论
  • 探索想法而非执行任务
  • 简单的一次性请求

开始使用

1

下载 Claude Desktop

从 Cowork 标签访问 Cowork。这是 macOS 桌面功能，面向 Max 计划用户提供预览

2

授予文件夹访问权限

指向你想要使用的目录。在写入之前会请求权限

3

连接你的工具

通过 MCP 连接器链接到你的实际工作环境，让 Cowork 更加强大

4

开始工作

使用提示构建器快速开始，或描述一个完整的任务让 Cowork 执行

使用技巧

💡 从提示构建器开始

打开 Cowork 时会看到"创建文件"、"处理数据"、"整理文件"、"准备一天"等建议，快速开始

🎯 用完整任务的角度思考

描述具有明确终点的任务。"分析这些文档并创建包含三大主题的幻灯片"比"帮我理解这项研究"更有效

📈 培养直觉

从明确界限的任务开始（"整理此文件夹"、"综合这些文档"），逐步扩大到更大项目
